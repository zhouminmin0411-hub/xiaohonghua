# 小红花小程序 - 项目实施清单（前端优先版）

## 项目概述

**目标**：6-8周内完成小红花小程序MVP版本开发并上线
**技术栈**：Node.js + Express + MySQL + 微信小程序
**开发模式**：个人全栈开发
**开发策略**：前端优先（Mock数据）→ 后端开发 → 联调上线
**部署平台**：阿里云ECS + 微信小程序

---

## 阶段一：项目准备与环境搭建（第1周）

### 1.1 账号与资源准备
- 注册微信小程序账号（个人/企业）
- 购买域名并提交备案（可延后，但备案需时间，建议提前）
- 阿里云服务器购买延后至第4周（后端开发前）
- SSL证书申请延后至部署前

### 1.2 开发环境搭建
- 安装Node.js（v16+）
- 安装微信开发者工具
- 配置Git版本控制
- 初始化项目代码仓库

### 1.3 技术选型确认
- 小程序UI组件库：Vant Weapp
- Mock数据方案：本地JSON文件（第1-3周使用）
- 后端框架：Express.js + Sequelize ORM（第4周实施）
- 数据库：MySQL（第4周实施）
- 定时任务：node-cron（第4-5周实施）

---

## 阶段二：数据模型设计（第1周）

### 2.1 数据结构设计文档

基于PRD第6章，设计数据模型结构（暂不建数据库）：

**User模型**
```
id, openid, role, parentPassword, nickname,
avatarUrl, childId, createdAt, updatedAt
```

**Task模型**
```
id, type, icon, title, description, reward,
createdByParentId, isActive, createdAt, updatedAt
```

**TaskRecord模型**
```
id, taskId, childId, status, reward,
receivedAt, completedAt, parentLikedAt, createdAt
```

**Reward模型**
```
id, title, cost, icon, type, isActive,
createdByParentId, createdAt, updatedAt
```

**RewardRecord模型**
```
id, rewardId, childId, cost, createdAt
```

**WeeklyPointConfig模型**
```
id, childId, weeklyAmount, dayOfWeek,
time, enabled, lastIssuedAt, updatedAt
```

**PointHistory模型**
```
id, childId, change, reason,
sourceType, sourceId, createdAt
```

### 2.2 Mock数据准备

创建 `mockData/` 目录，包含：
- `tasks.json` - 10条示例任务数据
- `rewards.json` - 8条示例奖励数据
- `taskRecords.json` - 已完成任务记录
- `user.json` - 用户信息
- `pointHistory.json` - 积分历史记录

## 关键里程碑

- **第1周末**：环境搭建完成，数据模型设计完成，Mock数据准备完成
- **第2周末**：小程序前端完成60%（首页、商城、设置页）
- **第3周末**：小程序前端完成100%（所有页面+动画）
- **第4周末**：后端项目搭建完成，数据库初始化完成，认证模块完成
- **第5周末**：后端API全部完成，定时任务完成
- **第6周末**：前后端联调完成，服务器部署完成
- **第7周末**：体验版上线，提交审核或已通过审核
- **第8周末**：正式上线并完成首轮优化
